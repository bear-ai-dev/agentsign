# AgentContract Landing UI Goal

Create a landing page inspired by https://www.agentmail.to/ but not copied.

Direction:
- Light mode, enterprise-grade version of AgentMail-style agent infrastructure marketing.
- Use AgentMail only for structural inspiration: agent-category headline, code/API hero surface, clear docs/CLI CTAs, enterprise trust posture, proof sections.
- AgentContract positioning: contract sending rails for agent-run onboarding. Agents send approved packets; people review and sign them. Do not imply agents sign contracts, draft legal terms, or give legal advice.
- Visual tone: crisp, technical, trustworthy, enterprise-ready, not playful, not generic SaaS cards. Copy should be plain and literal before it is clever.
- Must fit AgentContract: approved templates, uploaded PDFs, signing links, recipient/browser signing, signed PDFs, SHA-256 hashes, signed webhooks, audit trail.
